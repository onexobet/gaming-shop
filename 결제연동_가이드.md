# MORESEAT 카드결제 연동 가이드 (윈글로벌페이 + Netlify)

브라우저에는 PayKey가 절대 노출되지 않고, Netlify 서버 함수가 대신 윈글로벌페이와 통신하는 구조입니다.

## 결제 흐름
1. 장바구니 → **카드로 결제하기** → 주문자 정보 입력
2. 브라우저가 `/.netlify/functions/create-payment` 호출 (금액·상품명·주문자만 전달)
3. 서버 함수가 PayKey로 로그인 → JWT 발급 → 윈페이 결제요청 → **결제 URL만** 브라우저에 반환
4. 결제 팝업이 열리고, 사용자가 카드 결제
5. 완료 시 `payment-callback` 이 부모창에 결과 전달 → 팝업 닫힘
6. 성공하면 장바구니 비우고 `order-complete.html` 로 이동

## 파일 구성
- `netlify.toml` — 함수 디렉터리 설정
- `netlify/functions/_winpay.js` — 공통(로그인/설정)
- `netlify/functions/create-payment.js` — 결제 요청
- `netlify/functions/payment-status.js` — 결제 상태 조회
- `netlify/functions/payment-callback.js` — 결제 결과 수신(returnUrl)
- `order-complete.html` — 주문 완료/실패 안내 페이지

## 배포 방법 (드래그앤드롭이 아니라 Git 연결 권장)
서버 함수는 **폴더 드래그 배포로는 빌드가 안 되는 경우가 많습니다.** GitHub 연결 배포를 권장합니다.
1. 이 프로젝트 폴더를 GitHub 저장소에 올리기
2. Netlify → **Add new site → Import from Git** → 저장소 선택
3. Build command 비움, Publish directory `.` (netlify.toml에 이미 있음)
4. 배포 완료

> 드래그앤드롭으로 계속 하시려면, 압축 푼 폴더 안에 `netlify/functions/`, `netlify.toml`이 **함께** 포함된 채로 폴더째 올려야 합니다.

## 환경변수 설정 (가장 중요)
Netlify → Site settings → **Environment variables** 에 아래 4개를 등록하세요.
값을 넣기 전까지는 결제 버튼이 "자격증명이 설정되지 않았습니다" 메시지를 냅니다.

| 이름 | 값 | 설명 |
|---|---|---|
| `WINPAY_SERVER_URL` | `https://jh.winglobalpay.com` | 윈페이 서버 (테스트 기준) |
| `WINPAY_TMN_ID` | (발급받은 터미널 ID) | 윈글로벌페이에서 발급 |
| `WINPAY_PAYKEY` | (발급받은 PayKey) | **절대 코드/브라우저에 넣지 말 것** |
| `SITE_URL` | `https://내사이트.netlify.app` | 배포된 사이트 주소 (returnUrl 구성용) |

환경변수를 바꾼 뒤에는 **Deploys → Trigger deploy → Deploy site** 로 재배포해야 반영됩니다.

## 발급 요청 시 윈글로벌페이에 확인할 것
- 테스트 **터미널 ID(tmnId)** 와 **PayKey**
- **신용카드(CARD)** 결제수단 활성화
- 서버 주소가 `jh.winglobalpay.com` 이 맞는지(운영 주소가 다를 수 있음)
- `returnUrl` / `redirectUrl` 도메인 등록 필요 여부

## 보안 참고
- ✅ **금액 조작 방지 적용됨**: 서버(`create-payment.js`)가 `_prices.js`의 가격표로 결제금액을 다시 계산합니다. 브라우저가 보낸 금액은 무시합니다.
- ⚠ **가격을 바꾸면** `_prices.js`도 다시 만들어야 합니다. (상품 가격/추가는 `shop-data.js` 수정 후 알려주시면 제가 재생성해 드립니다.)
- 결제 성공은 `payment-callback` / `payment-status` 로 **서버에서 최종 확인**한 값만 신뢰하세요.
- 주문 데이터는 실제 DB에 저장하는 로직을 추가해야 합니다. (현재는 프론트 안내까지만)
